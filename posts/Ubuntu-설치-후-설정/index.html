Ubuntu 설치 후 설정

서론

  윈도우를 사용하다가 리눅스를 공부해보고 싶은 마음에 충동적으로 노트북에 우분투를 설치했습니다. 예전에 학교에서 배웠던 기억이 있는데, 몇 년전에 배워서 그런지 다 잊어버렸네요.

  우분투를 설치하는 것은 인터넷에서 본 블로그를 보고 따라 했습니다. 인터넷에 좋은 자료가 너무 많아서 설치과정은 차치하고 설치 후에 했던 것을 기록할 겸 포스팅하려 합니다.


목차

  • 업데이트

1. 기본설정

1-1. 미러서버 변경

 우분투를 설치할 때 기본으로 지정된 저장소를 카카오 미러 서버로 바꿔줬습니다.

 기본 저장소도 빠르지만 카카오 미러 서버가 비교적 더 빨라서 많은 사람들이 주 미러 서버를 카카오 서버로 설정하고 있다고 하네요.

 다음과 같은 명령어로 /etc/apt에 백업 파일을 만들고 카카오 미러 서버로 바꿀 수 있습니다. 

sudo sed -i'.backup' 's/kr.archive.ubuntu.com/mirror.kakao.com/g' /etc/apt/sources.list
  • 다른 미러 서버를 사용하고 싶으신 분들은 링크에 접속하신 뒤 Korea 항목에 있는 링크 중 하나로 사용하시면 됩니다.
  • sed 명령어를 사용하면 명령어로 간단하게 파일을 수정할 수 있습니다. 자세하게 알아보고 싶으신 분들은 링크를 참고하세요

1-2. 업데이트

 우분투에서는 주로 apt-get 명령어로 패키지를 관리합니다. 그리고 패키지에 대한 정보는 저장소로부터 업데이트할 수 있습니다. 따라서 저장소를 변경한 뒤, 패키지에 대한 목록을 업데이트합니다.

sudo apt-get update
sudo apt-get upgrade
  • 패키지에 대한 정보는 /var/lib/apt/lists 에서 확인할 수 있습니다.
  • apt-get update 는 사용 가능한 패키지들과 그 패키지들의 버전을 업데이트하는 명령어입니다.
  • apt-get upgrade 는 설치된 패키지들을 최신 버전으로 업그레이드 하는 명령어입니다.

1-3. 한글 설정

 우분투 설치 과정 중, 언어 설정 단계에서 한국어(101/104키 호환)를 추가했었는데 한/영 변환이 잘 안되더라구요.

그래서 한글과 관련된 입력기를 이것 저것 설치해보던 중, 이 글을 보고 해결할 수 있었습니다. 혹시 문제 있으신분들은 참고하세요.


1-4. 개발 도구 설치

 저는 vim, jdk, gcc, g++를 설치했습니다.

sudo apt-get install vim openjdk-14-* g++ gcc

1-5. 터치패드 비활성화

 한성 노트북에 우분투를 설치해서 사용하는데 한성에서는 리눅스에서 사용할 수 있는 드라이브를 제공해주지 않더라구요.

 윈도우에서는 터치패드 좌측 상단을 더블클릭하면 비활성화할 수 있는데 터치패드 비활성화가 안될 뿐만 아니라, 몇몇 Fn키가 먹히지 않았습니다. 그래서 터치패드 없이 사용하는게 낫겠다 싶어 우분투 설정에서 터치패드를 비활성화 했습니다.

우측 상단 설정 -> 마우스 및 터치패드 항목에서 끌 수 있습니다.


1-6. 하드디스크 마운트

 저는 우분투 OS가 설치된 저장장치 외에 외장하드 1개, HDD 1개를 사용하고 있습니다. 외장하드는 부팅시 자동으로 인식이 되는데 HDD는 안되더라구요. 폴더창에서 간편하게 마운팅을 할 수 있지만, 부팅할 때마다 매번 수동으로 마운트를 해줘야하기 때문에 HDD를 자동으로 마운트되도록 하겠습니다.

 먼저 특정 파티션을 마운트하려면 해당 파티션의 uuid를 알아야됩니다. uuid는 리눅스에서 파티션을 고유하게 식별하기 위해 사용되는 식별자이기 때문에 uuid를 알고 있으면 여러 개의 파티션 중에서 하나의 파티션을 특정할 수 있습니다.

  • 다음 명령어 중 하나로 파티션의 uuid를 확인합니다.
1) ls -l /dev/disk/by-uuid
2) sudo blkid | grep UUID
3) sudo lsblk -f | grep -v loop

 제가 마운트하려는 파티션은 위에서 3번째 파티션으로 해당 파티션의 uuid는 C820C4C020C4B72A입니다.


 리눅스에서 파일시스템에 대한 정보는 /etc/fstab파일에 저장되어 있습니다. fstab에서 하나의 파일 시스템은 다음과 같은 형식으로 한 줄로 표현됩니다. 따라서 다음 형식으로 마운트 할 파일시스템의 정보를 fstab에 추가합니다. 

<file system> <mount point> <type> <option> <dump> <pass>
필드들의 의미
  1. file system: 설정하려는 파일시스템의 uuid입니다.
  2. mount point: 파일시스템이 마운트될 지점입니다. 원하는 경로를 지정해줍니다.
  3. type: 파일시스템의 타입으로 uuid를 확인하는 명령어에서 파일시스템도 확인할 수 있습니다.
  4. option: 사용하려는 환경에 맞는 옵션을 지정해줍니다.
                옵션에 대한 모든 정보는 “man mount” 명령어로 확인할 수 있습니다.
               - default: rw, suid, dev, exec, auto, nouser, async 옵션을 사용합니다.
               - auto: 부팅시 파일시스템을 마운트합니다.
               - noauto: 부팅시 파일시스템이 마운트되는 것을 방지합니다.
               - users: root가 아닌 user도 마운트할 수 있습니다.
               - nousers: root만 마운트할 수 있습니다.
               - ro: read only로 읽기 전용 옵션입니다.
               - rw: read write로 읽고 쓰는 동작을 허용합니다.
  5. dump: (0|1) 중 하나를 선택할 수 있으며 덤프가 필요한 파일시스템이면 1을 선택합니다.

  6. pass: (0|1|2) 중 하나를 선택할 수 있습니다.
               부팅 시 fsck 명령어로 파일시스템을 점검할 때 pass 필드로 검사 순서를 제어할 수 있습니다.
               0은 파일 시스템을 검사하고 싶지 않을 때 지정합니다.
               1은 루트 파일시스템에 지정합니다.
               2는 그 외의 파일시스템에 사용합니다.


  • 다음 명령어로 /etc/fstab에 마운트하려는 파일시스템의 정보를 삽입합니다.
sudo sed -i '$aUUID=C820C4C020C4B72A /mnt/HDD1 ntfs defaults 0 0' /etc/fstab

저는 uuid가 C820C4C020C4B72A인 파일시스템을 defaults 옵션을 사용해서 /mnt/HDD1에 마운트했습니다.


  • 마지막으로 수정한 fstab의 정보를 반영하고, 마운트 포인트의 심볼릭 링크를 만들어줍니다.
1) sudo mount -a	# fstab에 언급된 모든 파일시스템을 마운트합니다.
2) ln -s /mnt/HDD1 [path]	# 마운트포인트 /mnt/HDD1에 대한 심볼릭 링크를 path에 생성합니다.

2. 테마

2-1. tweak

 tweak는 우분투의 GUI 테스크탑 환경인 그놈(GNOME)의 구성요소를 변경할 수 있는 도구입니다. tweak로 폰트나 아이콘, dock 같은 기본 구성요소를 변경할 수 있습니다.

  • 다음 명령어 중 하나로 tweak를 설치할 수 있습니다.
sudo apt-get install gnome-tweaks
  or
sudo apt-get install gnome-tweak-tool

패키지를 설치하면 애플리케이션에 기능 개선 항목이 생성됩니다.

우선 설치만 하고 다음 테마를 적용하면서 어떤 설정을 할 수 있는지 알아보겠습니다.


2-2. Dock

 우분투의 Dock은 애플의 맥 OS에 있는 Dock과 같은 기능을 제공합니다. 그런데 맥 OS와 다르게 우분투에서 제공하는 기본 Dock의 레이아웃은 조금 투박합니다. 그래서 테마를 적용하시는 분들을 보면 대부분 Dock을 바꾸시더라구요.

 우분투에서 Dock에 적용할만한 대표적인 테마는 Plank와 Dash to dock 두 가지가 있습니다. 저는 두 가지 독을 전부 설치해서 다음처럼 쓰고 있습니다.

1번은 Dash to dock을 적용했고, 2번은 Plank를 적용했습니다.

- Dash to dock

 Dash to dock은 파이어폭스나 크롬 브라우저에서 Dash to Dock extension로 접속한 뒤 간단하게 설치할 수 있습니다.

  • 위의 링크에 접속한 뒤 OFF라고 써진 버튼을 클릭해서 ON으로 바꿔줍니다.

설치가 끝나면 2-1에서 설치했던 tweaks를 통해 독을 조정할 수 있습니다.
  1. 유틸리티에서 기능 개선 앱을 실행합니다.
  2. 확장 탭을 클릭한 뒤 아래 이미지에서 Ubuntu dock를 제외한 나머지 버튼을 활성화합니다.
  3. Ubuntu dock 버튼은 비활성화합니다.

 각각의 버튼을 활성화 또는 비활성화 하면 독이 바뀐 모습을 볼 수 있습니다. 독의 레이아웃을 조정하려면 위 이미지에서 Dash to dock 항목 오른쪽에 메뉴 버튼을 클릭한 뒤, 활성화된 창에서 세부적인 사항을 조정할 수 있습니다.

- Plank

다음 명령어로 Plank를 설치합니다.

sudo apt-get install plank

 plank를 설치하고 나서 다음 명령어로 레이아웃을 조정할 수 있습니다. 위의 명령으로 plank를 설치하면 앱이 생성되는데, 생성된 앱을 클릭하면 실행이 안되더라구요. 그래서 명령어로 설정창을 실행합니다. 
plank --preferences

저는 다음처럼 설정했습니다. ![plank1.png](/media/plank1.png)

 도크릿 탭에서 아이콘을 클릭하면 plank 독에 클릭한 아이콘이 추가됩니다. Desktop 아이콘은 바탕화면을 표시하는 아이콘입니다. 윈도우에서 window + D 커맨드와 동일한 기능을 합니다.

 plank에 있는 앱을 우클릭한 뒤 나타나는 플랭크에 두기 항목을 활성화하거나 비활성화해서 앱을 독에 추가하거나 지울 수 있습니다. 만약 앱을 지우고 싶은데 플랭크에 두기가 없다면 다음 경로에서 지우고 싶은 앱을 지우면 됩니다.

~/.config/plank/dock1/launchers/

2-3. 테마

 전체적인 테마를 변경하기 위해서 여기에서 User Themes를 설치합니다. User Themes를 설치한 뒤 기능 개선 앱에서 테마를 변경할 수 있습니다.

 테마는 여기에서 원하는 테마를 다운로드하면 됩니다. 저는 다음과 같은 테마를 적용했습니다.

Theme: Orchis-light
Icon: Numix Circle Icon Theme

 홈에 `.themes`폴더를 생성한 뒤, `~/.themes`에 위의 링크에서 다운받은 테마를 옮깁니다. 
mkdir ~/.themes
mv Ordchis-light ~/.themes

 그 다음 기능개선 앱에서 테마를 변경합니다. 모양새 탭에서 레이아웃, 커서, 아이콘 등의 테마를 변경할 수 있습니다.


2-4. 쉘

 z쉘은 각종 플러그인을 적용할 수 있고 경로 추론 및 명령어 선택 등 다양한 기능을 제공해서 기본 쉘인 bash보다 편의성이 더 좋다고 합니다. 또한 oh-my-zsh라는 툴로 z쉘의 사용성을 개선할 수 있어서 많은 분들이 z쉘을 사용하시더라구요. 그래서 저도 한번 써봤습니다.

 다음 명령어로 z쉘과 oh-my-zsh를 설치하고 기본쉘로 변경합니다. (zsh)

1) sudo apt install zsh -y
2) chsh -s /usr/bin/zsh
3) sh -c "$(curl -fsSL https://raw.github.com/robbyrussell/oh-my-zsh/master/tools/install.sh)"

 zsh 테마는 링크 및 링크에서 확인하실 수 있으며, 다음 순서로 다운받은 테마를 적용할 수 있습니다.

  1. 확장자가 *.zsh-theme인 파일을 ~/.oh-my-zsh/custom/themes로 옮깁니다.
  2. ~/.zshrc 파일에서 ZSH_THEME="{테마명}"인 라인의 테마명을 다운받은 테마명으로 수정합니다.
  3. source ~/.zshrc 명령어로 수정사항을 적용합니다.

 z쉘의 외부 플러그인도 ~/.oh-my-zsh/custom/plugins/에 옮기고 .zshrc를 수정해서 사용할 수 있습니다.


2-5. 글꼴

 적용하고 싶은 글꼴의 ttf파일을 다운받고 실행해서 설치합니다. 저는 Naver D2를 받았습니다.

 그 다음, 기능개선 앱의 글꼴 탭에서 다운받은 글꼴로 변경합니다.


2-6. 바탕화면 아이콘 제거

 바탕화면에는 있지만 바탕화면 폴더에서 제거할 수 없는 아이콘(ex.쓰레기통)을 제거하려면 dconf 패키지가 필요합니다. 다음 명령으로 dconf를 설치한 뒤 실행합니다.

1) sudo apt-get install dconf-*
2) dconf-editor

  아래의 화면에서 show-home 및 show-trash의 값을 거짓으로 변경하면 바탕화면에서 쓰레기통과 유저 폴더를 제거할 수 있습니다. ![dconf](/media/dconf.png) > 어플리케이션 테이블의 'dconf 편집기'로도 실행할 수도 있다.
